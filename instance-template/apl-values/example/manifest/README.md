# `example/manifest/` — per-environment manifests

This directory holds the **Argo CD Applications and raw Kubernetes manifests**
for an environment that are *not* expressed through apl-core's `values.yaml`.
apl-core's bundled Argo CD syncs them via the BYO-Git repo you configure under
`otomi.git` in [`../values.yaml`](../values.yaml). `kustomization.yaml` is the
sync entrypoint; `llz env add <env> --template-env example`
clones this whole tree to `apl-values/<env>/manifest/` and swaps the env
identity tokens.

## What the template ships here

Two kinds of content:

1. **Platform glue (shipped)** — the cluster-identity-coupled raw manifests
   every environment needs that are *not* apl-core values and *not* first-party
   Helm charts:
   - `external-secrets/` — the `openbao` **ClusterSecretStore** (the ESO↔OpenBao
     binding every ExternalSecret depends on) + the ESO namespace NetworkPolicies.
   - `cert-manager/openbao-bootstrap-ca.yaml` — the stable self-signed
     `openbao-ca` ClusterIssuer that signs the `llz-openbao-platform` chart's
     `openbao-tls` cert (the chart depends on this issuer existing).
   - `openbao/openbao-cert-watcher.yaml` — restarts OpenBao on TLS rotation.
   - `linode-volume-labeler/` — the relabeler CronJob/RBAC/NP/ConfigMap (its
     namespace + token Secret are TF-owned in `terraform-iac-bootstrap/cluster-bootstrap`).
   - `observability/` — Loki object-store ExternalSecret + platform
     `PrometheusRule` alert rules (`openbao-alerts`, `support-plane-alerts`).
   - `harbor/harbor-registry-s3-externalsecret.yaml` — Harbor S3 registry creds.
   - `dns/` — Let's Encrypt ClusterIssuers + the DNS-01 solver ExternalSecret +
     the cert-manager-webhook-linode Application. **Applied separately** by
     `.github/workflows/bootstrap-dns.yml` (NOT in `kustomization.yaml`) because
     it needs the operator-supplied Linode DNS token first.

   The Argo CD Applications that wrap the first-party `llz-*` charts
   (external-secrets, argo-workflows/events, cert-automation, eso-cert-watcher,
   openbao, linode-internal-cidr-firewall) are **not** here — they are
   generated by the `llz-argo-bootstrap-apps` app-of-apps chart in
   `kubernetes-charts/`.

2. **Your workloads (you add)** — the instance's own application
   Deployments/Services/Ingress and any product-specific Argo CD Applications.
   The template ships none of these.

## Per-env tokens

Values written `REPLACE_PER_ENV` (e.g. the Let's Encrypt registration email)
are filled in per environment. `REGION_SHORT` on the volume-labeler is swapped
by `llz env add` via `linode-volume-labeler-region-patch.yaml`.
